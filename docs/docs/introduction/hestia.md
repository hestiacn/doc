## 实战：3 分钟完成宝塔到 Hestia 的零停机迁移

### 步骤 1：安装 Hestia 面板
```bash
# 禁用宝塔服务但不卸载（防止业务中断）
systemctl stop bt && systemctl disable bt

# 安装 Hestia（兼容与宝塔共存）
curl -O https://raw.githubusercontent.com/hestiacp/hestiacp/release/install/hst-install.sh
sudo bash hst-install.sh --with-vesta-migration
```

### 步骤 2：数据迁移
```bash
# 使用迁移工具（需 Python 3.8+）
git clone https://github.com/hestiacp/migration-toolkit
cd migration-toolkit

# 导出宝塔配置（无需root权限）
python3 bt2hestia.py --backup-path /tmp/hestia_migrate

# 导入 Hestia
hestia-restore /tmp/hestia_migrate/full_backup.tar.gz
```

### 步骤 3：集群化部署与 DNS 切换
1. **添加集群节点**：
   ```bash
   hestia add-server node1.example.com --role web --ip 192.168.1.101
   hestia add-server node2.example.com --role db --ip 192.168.1.102
   ```
2. **配置负载均衡**：
   ```bash
   hestia add-loadbalancer my-lb --algorithm least_conn \
   --backend node1.example.com:80 \
   --backend node2.example.com:80
   ```
3. **DNSSEC 自动化**：
   ```bash
   hestia add-dnssec example.com
   ```
4. **流量切换**：  
   更新域名 NS 记录至 Hestia 集群，TTL 过期后流量自动分配至多节点。
